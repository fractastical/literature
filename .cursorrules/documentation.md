# Documentation Requirements

## Documentation Structure

### Folder-Level Documentation
- **Every folder level must have accurate and complete AGENTS.md and README.md**
- AGENTS.md provides comprehensive module documentation
- README.md provides usage and quick reference
- Both files must be maintained and kept current

### AGENTS.md Requirements
- Comprehensive module documentation
- Architecture and design decisions
- API reference and usage examples
- Integration guidelines and patterns
- Troubleshooting and common issues

### README.md Requirements
- Quick start and usage guide
- Key features and capabilities
- Installation and setup instructions
- Basic examples and common workflows
- Links to detailed documentation

## Documentation Style

### "Show Not Tell" Principle
- Documentation should demonstrate, not just describe
- Include concrete examples and code samples
- Show actual usage patterns and workflows
- Provide real-world scenarios and use cases

### Understated and Factual
- Keep documentation understated and factual
- Avoid marketing language or hyperbole
- Focus on accurate, clear descriptions
- Let functionality speak for itself

### Clear and Interpretable
- Write documentation that is easy to understand
- Use clear, concise language
- Structure information logically
- Make documentation accessible to all skill levels

## Code Documentation

### Code Comments
- Write clearly-commented, interpretable code
- Use comments to explain "why" not "what"
- Document complex logic and algorithms
- Explain design decisions and trade-offs

### Method Documentation
- Document all public methods and classes
- Include purpose, parameters, return values, and exceptions
- Provide usage examples where helpful
- Note any side effects or important behaviors

### Module Documentation
- Document module purpose and responsibilities
- Explain module architecture and design
- Describe integration points and dependencies
- Provide usage examples and common patterns

## Documentation Quality

### Accuracy and Completeness
- Ensure documentation is accurate and complete
- Keep documentation synchronized with code
- Verify all examples work correctly
- Update documentation when code changes

### Clarity and Organization
- Organize documentation logically
- Use clear section headers and structure
- Provide navigation and cross-references
- Make information easy to find

### Examples and Use Cases
- Include practical examples
- Show common use cases and patterns
- Demonstrate error handling
- Provide troubleshooting guidance

## Documentation Maintenance

### Keeping Documentation Current
- Update documentation when code changes
- Review documentation for accuracy regularly
- Remove obsolete information
- Add documentation for new features

### Documentation Review
- Review documentation for completeness
- Verify examples are correct and current
- Check for clarity and organization
- Ensure documentation follows style guidelines

## Special Documentation Requirements

### API Documentation
- Document all public APIs comprehensively
- Include parameter descriptions and types
- Document return values and exceptions
- Provide usage examples for each API

### Configuration Documentation
- Document all configuration options
- Explain default values and behavior
- Provide configuration examples
- Note any environment-specific requirements

### Workflow Documentation
- Document complete workflows and pipelines
- Explain orchestration and coordination
- Provide step-by-step guides
- Include troubleshooting for common issues

### Integration Documentation
- Document integration points and patterns
- Explain how modules work together
- Provide integration examples
- Note any integration requirements or constraints

## Documentation Standards

### Markdown Format
- Use standard Markdown formatting
- Follow consistent formatting conventions
- Use code blocks for code examples
- Include proper heading hierarchy

### Code Examples
- Provide working code examples
- Use realistic scenarios
- Include error handling in examples
- Show both simple and advanced usage

### Cross-References
- Link to related documentation
- Reference other modules and components
- Provide links to external resources
- Maintain consistent linking patterns

## Documentation for Different Audiences

### Developer Documentation
- Focus on implementation details
- Explain architecture and design decisions
- Provide API references
- Include troubleshooting guides

### User Documentation
- Focus on usage and workflows
- Provide quick start guides
- Explain common tasks
- Include examples and tutorials

### Maintenance Documentation
- Document system architecture
- Explain maintenance procedures
- Provide troubleshooting guides
- Include backup and recovery procedures
